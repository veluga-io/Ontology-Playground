# 위젯 보안 분석을 내장

## 위협 모델

인제된 위젯은 카탈로그에서 오노토미 데이터를 로드하고 타사 사이트에서 렌더링합니다. 커뮤니티 사용자는 푸리 요청을 통해 카탈로그에 오노토미를 제출할 수 있습니다.

**질문:** 악성 카탈로그 제출이 포함 요소 위젯을 사용하는 사이트에 코드를 삽입할 수 있습니까?

**답변:** 아니요. 내장된 위젯은 설계상 안전합니다.

> 범위 메모: 이 문서는 **임베드 위젯** 위협 모델을 다룹니다. 배우다
> 기사 마크다운 렌더링(`/#/learn`)은 별도의 소독 경계선이 있습니다.
> 컴파일 파이프라인(`scripts/compile-learn.ts`)에 있으며 다음의 일부가 아닙니다.
> 런타임 공격 표면을 내장한다.

## 데이터 흐름

형이상학적 데이터(엔터티 이름, 설명, 아이콘, 색상, 속성 이름, 관계 이름)는 세 가지 렌더링 경로로 흐릅니다.

1. **React JSX** - 엔터티 이름, 설명, 속성 배지, 관계 세부 정보
2. **세포스케이프 캔버스** - 그래프 노드 레이블, 에지 레이블, 노드 색상
3. **RDF 세리얼라이저** - RDF 소스 탭에 표시되는 XML 출력

## 각 경로가 안전한 이유

### 1. React JSX 탈출

모든 사용자 제어된 문자열은 HTML 엔터티를 자동으로 이스케이프하는 JSX 대입(`{variable}`을 통해 렌더링됩니다. 인베드 위젯의 어디에서도 **`dangerouslySetInnerHTML`**을 사용하지 않습니다.

`<script>alert(1)</script>`과 같은 악성 엔터티 이름은 문자 그대로 텍스트로 표시됩니다.

```
<script>alert(1)</script>
```

### 2. 사이토스코이프 캔버스 렌더링

Cytoscape.js HTML5 Canvas `fillText()` API를 사용하여 그래프 레이블을 렌더링합니다, 이 API는 텍스트를 픽셀로 그리는데, HTML 파싱하거나 스크립트를 실행하지 않습니다. 악성 레이블 문자열은 캔버스에서 문자 그대로의 텍스트로 나타납니다.

### 3. RDF 세리얼라이저 XML 이스케이프

`serializeToRDF()` 함수는 대입 전에 모든 온톨로지적 값을 `escapeXml()`를 통해 전달하며, 이는 `&`, `<`, `>`, `"`, `'`을 XML 엔터티로 대체합니다. 그런 다음, 이스케이프된 출력은 JSX 대입을 통해 `<pre>` 태그 내에서 렌더링되어 이중 이스케이프가 제공됩니다.

## 추가 보호

| 레이어 | 보호 |
|---|---|
| **카탈로그 빌드** | 디렉토리 이름은 `SAFE_SLUG_RE`(알파벳 및 숫자 + 하이픈만)에 대해 검증되었으며, 심링은 거부됨 |
| **RDF 검증** | `validate-rdf.ts` 스크립트는 CI 동안 엔터티 구조의 검증 |
| **PR 검토** | 커뮤니티 제출물은 병합 전에 인간의 검토가 필요합니다 |
| **위험한 DOM 싱크 없음** | 포함 코드에 `dangerouslySetInnerHTML`, `innerHTML`, `document.write`, 또는 `eval`이 없음 |

## 색상 필드

엔터티 `color` 값은 Cytoscape의 캔버스 렌더러에서만 `background-color` 데이터로 소비됩니다. 유효하지 않거나 악성인 색상 값은 빈 또는 기본 색상의 원형을 생성합니다. 캔버스는 CSS 처리하지 않기 때문에 CSS 주입 벡터가 없습니다.

## `data-ontology-url` 고려 사항

`data-ontology-url` 속성은 임의의 URL에서 엔트로지를 로드할 수 있도록 합니다. 이는 카탈로그 제출자가 아니라 **페이지 작성자**(HTML 작성하는 사람)가 지정합니다. 가져온 데이터는 위에서 설명한 것과 동일한 안전한 React/Canvas 렌더링 경로를 통해 처리됩니다. 인제 넣은 위젯은 가져온 콘텐츠를 실행하거나 평가하지 않습니다. JSON 또는 RDF/XML 파싱하고 결과 엔트로지 구조를 렌더링하는 데만 사용됩니다.

## 요약

| 공격 벡터 | 완화되었는가? | 어떻게 |
|---|---|---|
| 엔터티/관계 이름에 의한 XSS | 예 | React JSX 자동 이스케이프 |
| 그래프 레이블을 통한 XSS | 예 | 캔버스 `fillText()`는 HTML 파싱하지 않습니다 |
| RDF 출력을 통한 XSS | 예 | `escapeXml()` + JSX 이스케이프 |
| 색상 통하여 CSS 주입 | 예 | DOM CSS 아닌 캔버스 렌더러 |
| 카탈로그 ID를 통한 경로 탐색 | 예 | `SAFE_SLUG_RE` 검증 |
| `data-ontology-url`을 통한 코드 주입 | N/A | 페이지 작성자가 설정한 URL, 데이터 안전하게 렌더링 |
