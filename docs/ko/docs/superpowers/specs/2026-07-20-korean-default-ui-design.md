# 한국어 기본 UI 설계

## 목표

`npm run dev` 실행 시 한국어를 기본 언어로 사용하되 영어 선택지를 유지하고, 온톨로지 도메인 데이터와 기술 식별자는 영어로 보존합니다.

## 사용자에게 보이는 동작

- 처음 방문한 사용자는 애플리케이션 셸, 탐색, 대화상자, 안내 및 학습 페이지를 한국어로 봅니다.
- 헤더는 데스크톱과 모바일에서 `한국어 / English` 언어 선택기를 제공합니다.
- 선택한 언어는 `localStorage`에 저장되고 다음 방문 시 복원됩니다.
- 온톨로지 이름, 엔터티/속성/관계 이름, 샘플 값, 쿼리 구문, RDF, JSON 키, 파일 형식 및 기타 기술 식별자는 변경하지 않습니다.
- 선택기를 통해 영어를 계속 완전히 사용할 수 있습니다.

## 아키텍처

i18n 의존성을 추가하는 대신 저장소 내부에 작고 타입이 지정된 지역화 계층을 사용합니다. 로케일 스토어는 `ko`와 `en`을 제공하고 `ko`를 기본값으로 사용하며, 저장된 값을 검증하고 React 컴포넌트에 번역 함수를 제공합니다. 번역 키는 기능별로 그룹화하여 누락된 키가 TypeScript 컴파일 단계에서 실패하도록 합니다.

주요 애플리케이션 작업 흐름의 사용자용 문자열은 로케일 사전으로 이동합니다. 데이터에서 가져온 레이블은 그대로 렌더링합니다. 퀘스트와 생성된 안내 문구에는 지역화 템플릿을 사용하지만, 보간되는 온톨로지 용어는 원래 언어로 유지합니다.

## 학습 콘텐츠

빌드 스크립트는 두 매니페스트를 컴파일합니다.

- `content/learn`에서 `public/learn.en.json`으로
- `docs/ko/content/learn`에서 `public/learn.ko.json`으로

`LearnPage`는 활성 로케일에 맞는 매니페스트를 로드합니다. 기본적으로 한국어를 먼저 요청하며, 지역화된 매니페스트를 불러오지 못하면 영어로 폴백하고 두 요청이 모두 실패한 경우에만 지역화된 오류를 표시합니다. 외부 소비자의 호환성이 깨지지 않도록 기존 `public/learn.json`을 호환성 산출물로 유지합니다.

## 범위

주요 셸과 핵심 학습/설계 작업 흐름을 번역합니다. 대상은 헤더와 모바일 메뉴, 명령 팔레트와 탭, 카탈로그/갤러리, 학습 페이지와 퀴즈, 도움말/정보/환영/투어 대화상자, 검사기, 퀘스트 패널, 쿼리 플레이그라운드, 경로 찾기, 디자이너 컨트롤, 주요 가져오기/내보내기 및 데이터 원본 대화상자입니다.

온톨로지 카탈로그 데이터, 가져온 온톨로지 콘텐츠, 샘플 바인딩, 코드 예제 내부의 기술 어휘 또는 직렬화된 출력은 번역하지 않습니다.

## 테스트

다음 항목에 집중한 테스트 주도 개발을 따릅니다.

- 한국어가 기본 로케일인지 확인합니다.
- 저장된 유효한 영어 값이 복원되고 잘못된 값은 한국어로 폴백하는지 확인합니다.
- 언어 선택기가 로케일을 변경하고 저장하는지 확인합니다.
- `LearnPage`가 로케일별 매니페스트를 요청하고 영어로 폴백하는지 확인합니다.
- 학습 컴파일러가 한국어와 영어 매니페스트를 생성하는지 확인합니다.
- 대표적인 주요 UI 레이블이 한국어로 렌더링되면서 온톨로지 이름은 변경되지 않는지 확인합니다.

구현 중에는 집중 테스트를 실행한 다음 전체 테스트 모음과 프로덕션 빌드를 실행합니다. 마지막 공백 검사는 `git diff --check`로 수행합니다.
