## 1. RDF 수입/내보내기(전체 테스트 범위 포함)


현재의 RDF 내보내는 것은 인라인입니다. `ImportExportModal.tsx` 그리고 아무것도 없다 RDF **수입**. RDF 첫 번째 등급의 서열화 형식으로 변해야 한다.


### 1.1 추출물 RDF 서열화 모듈
- [x] 생성 `src/lib/rdf/serializer.ts` - 기존 것을 옮기다 `exportAsRDF()` 밖의 논리 `ImportExportModal.tsx` 순수한 함수로 `serializeToRDF(ontology, bindings) → string`
- [x] 생성 `src/lib/rdf/parser.ts` — 구현하다 `parseRDF(rdfXmlString) → { ontology, bindings }` 경량 XML 파서 사용(브라우저 `DOMParser`; 무거운 의존성이 없음)
- [x] 지원 OWL 클래스 → 엔터티 유형, 데이터 유형 특성 → 특성, 객체 특성 → 관계
- [x] 왕복 신뢰성: `parse(serialize(ontology))` 동등한 논리학을 생산해야 한다


### 1.2 와이어 RDF UI로 가져오기
- [x] 안에 `ImportExportModal.tsx`, 받아들이다 `.rdf` 그리고 `.owl` 파일 입력에 있는 파일들
- [x] 확장자 및/또는 XML 프로로그를 통해 형식을 감지하고, 다음으로 이동합니다. RDF 구문 해석 프로그램
- [x] 다음 조건이 충족되는 경우 인라인에서 검증 오류를 표시합니다. RDF 잘못 형성되어 있다


### 1.3 완전한 테스트 배터리
- [x] 비테스트를 설정하세요 (`vitest`, `@testing-library/react`, `@testing-library/jest-dom`)
- [x]의 단위 테스트 `serializer.ts`:
  - 빈 온톨로지
  - 모든 속성 유형(문자열, 정수, 소수점, 날짜 등)을 포함한 논리학
  - 관계 속성을 가진 논리학
  - XML 특수 문자 이스케이프(& < > " ')
  - 코멘트에서 데이터 바인딩 보존
- [x]의 단위 테스트 `parser.ts`:
  - 유효하다 RDF/OWL 입력 → 올바른ontology 모양 수정
  - 필수 필드가 누락됨 → 설명적 오류
  - 네임스페이스 처리(사용자 정의 접두사, 기본 네임스페이스)
  - 형식 불량 XML → 우아한 오류
- [x] 왕복 테스트: 각 샘플 오노토지에 대해 시퀀싱 → 파싱 → 심층 동등성 검사 수행 `sampleOntologies.ts` 그리고 `cosmicCoffeeOntology`
- [x] 통합 테스트: 가져오기 RDF 모달을 통해 파일을 제출하고, 스토어 상태를 확인합니다.
- [x] 추가 `"test": "vitest run"` 그리고 `"test:watch": "vitest"` 에 `package.json` 스크립트

## 4. 내장 가능한 온톨로지적 위젯


외부 페이지(블로그, 튜토리얼, 문서)에 대화형 오노토미 뷰어를 내장할 수 있도록 허용하세요. 이는 CodePen이나와 유사합니다. GitHub Gist는 작업을 포함합니다.


### 4.1 독립형 포함 빌드
- [x] 별도의 Vite 엔트리 포인트를 생성하세요 `src/embed.tsx` 그것은 최소한의, 자체적으로 포함된 논리학 뷰어를 생성합니다:
  - 사이토스코이프 그래프 시각화(읽기 전용)
  - 클릭 시 엔터티/관계 검사기
  - 그래프 뷰와 사이에서 전환하기 위해 탭을 누르세요. RDF 소스 보기
  - 다음을 통해 오노토미 데이터를 수락합니다:
    - `data-ontology-url` 속성(URL로 `.rdf` 또는 `.json` 파일)
    - `data-ontology-inline` 속성(인라인 JSON, base64-인코딩됨)
    - `data-catalogue-id` 속성(게시된 곳에서 로드됨 `catalogue.json`)
- [x] 단일 JS로 구축 + CSS 묶음: `ontology-embed.js` + `ontology-embed.css`
- [x] 포함 대상에 대한 Vite 빌드 구성을 추가하세요:

  ```ts
  // vite.config.embed.ts
  build: {
    lib: { entry: 'src/embed.tsx', formats: ['iife'], name: 'OntologyEmbed' },
    rollupOptions: { output: { assetFileNames: 'ontology-embed.[ext]' } }
  }
  ```
- [x] 번들 크기를 150KB gzip 이하로 유지하세요(Cytoscape는 ~90KB gz이며, 이를 고려해야 합니다)


### 4.2 API 통합 및 사용
- [x] 외부 페이지의 사용 패턴:

  ```html
       data-theme="dark"
       data-height="500px">
  </div>
  <script src="https://<site>/ontology-embed.js"></script>
  ```

- [x] 각 온톨로지에 대한 메인 앱 갤러리에서 "삽입 코드 복사" 버튼을 제공하세요.

## 5. 보완적인 기능들


이러한 요소들은 커뮤니티 학습 리소스의 전반적인 경험을 향상시킵니다.


### 5.1 심층 링크 / URL 라우팅
- [x] 클라이언트 측 라우팅 추가(예: 경량 해시 기반 라우터)
- [x] 지원 경로:
  - `/#/` - 홈(현재 기본 오노토미)
