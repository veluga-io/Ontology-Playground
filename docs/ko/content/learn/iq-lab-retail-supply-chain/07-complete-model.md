## 마지막 두 엔터티

마지막 두 엔터티 유형은 소매 수명 주기의 루프를 닫습니다.

- **프로모션** — 판매를 촉진하는 마케팅 캠페인
- **반품** — 주문과 제품 모두에 연결되어 반품되는 상품

## 프로모션

특정 제품을 타겟팅하는 마케팅 캠페인:

| 부동산 | 유형 | 식별자? |
|---|---|---|
| `promotionId` | 문자열 | `promotionName` |
| `isActivePromotion` | 문자열 | |
| ✓ | 부울 | |

`isActivePromotion` 플래그를 사용하면 현재 캠페인에 대한 필터링이 가능합니다. 즉, "반환율이 높은 제품과 관련된 활성 프로모션은 무엇입니까?"와 같은 질문입니다. 프로모션 → 제품 ← 반품을 순회합니다.

## 반품

주문 및 제품으로 다시 연결되는 반품된 품목:

| 부동산 | 유형 | 식별자? |
|---|---|---|
| `returnId` | 문자열 | `returnDate` |
| `reason` | 날짜 | |
| ✓ | 문자열 | |

## 새로운 관계

세 가지 최종 관계:

- **PromotionForProduct** — `Promotion` → `Product`(다대일) 프로모션이 대상으로 하는 제품입니다.

- **ReturnForOrder** — `Return` → `Order`(다대일) 반품이 연결된 주문입니다.

- **ReturnOfProduct** — `Return` → `Product`(다대일) 어떤 제품이 반품되었는지입니다.

## 완전한 모델이 가능하게 하는 것

전체 온톨로지가 구축되면 자연스럽게 대답할 수 있는 질문의 예는 다음과 같습니다.

| 질문 | 그래프 경로 |
|---|---|
| 어떤 프로모션이 반품을 유도했나요? | 프로모션 → 제품 ← 반품 |
| 반품된 제품의 재고는 얼마나 됩니까? | 반품 → 상품 ← 재고 → 창고 |
| 수요가 많은 지역에 서비스를 제공하는 항공사는 어디인가요? | DemandSignal → 지역 ← 매장; 배송 → 운송업체 |
| 어떤 고객이 판촉 제품을 주문했습니까? | 고객 ← 주문 → OrderLine → 제품 ← 프로모션 |
| 제품 부족 현상은 어떻게 예상되나요? | 재고 → 제품 ← 예측 |

이들 각각에는 복잡한 다중 테이블 SQL 조인이 필요합니다. 온톨로지를 사용하면 그래프 순회로 표현되며 Fabric IQ에서는 자연어 데이터 에이전트가 온톨로지 구조에서 이에 응답할 수 있습니다.

## GQL 쿼리 예

첫 번째 질문인 "어떤 프로모션이 반품을 유도했습니까?"는 다음과 같습니다. — GQL에서 살펴보겠습니다.

```gql
MATCH (r:Return)-[:ReturnOfProduct]->(p:Product)<-[:PromotionForProduct]-(promo:Promotion)
WHERE promo.isActivePromotion = true
RETURN promo.promotionName, p.name, r.reason
```

GQL 패턴은 설계한 온톨로지 관계를 직접 반영합니다. 모델과 쿼리 사이에 임피던스 불일치가 없습니다.

## 우리가 만든 것

6단계에 걸쳐 우리는 점진적으로 완전한 온톨로지를 구축했습니다.

| 단계 | 추가된 엔터티 | 누적 | 핵심 개념 |
|---|---|---|---|
| 1 | 고객, 주문, 상품 | 3 | 엔터티 유형, 식별자, 카디널리티 |
| 2 | OrderLine, ProductCategory | 5 | 엔터티, 계층 연결 |
| 3 | 지역, 매장 | 7 | 지리적 구조, 부울 속성 |
| 4 | 배송, 캐리어, 창고 | 10 | 허브 엔터티, 도메인 간 연결 |
| 5 | 재고, 예측, 수요신호 | 13 | 소스 간 통합, 기획 데이터 |
| 6 | 프로모션, 반품 | 15 | 루프 닫기, GQL 쿼리 |

## 주요 내용

1. **작게 시작하여 점진적으로 성장** - 세 가지 요소만으로도 가치를 창출할 수 있습니다.
2. **엔터티 연결**은 다대다 속성 문제를 해결합니다.
3. **허브 엔터티**(예: 배송)는 다양한 도메인을 연결합니다.
4. **교차 소스 통합**이 핵심 가치입니다. 하나의 온톨로지, 다중 데이터 엔진
5. **그래프 순회**는 복잡한 SQL 조인을 직관적인 경로 패턴으로 대체합니다.
6. **온톨로지는 API입니다** - GQL 쿼리와 데이터 에이전트 질문은 모두 동일한 구조를 따릅니다.

```quiz
Q: 전체 소매 온톨로지에서 "어떤 프로모션이 반품을 유도했습니까?"라는 질문을 어떻게 표현하시겠습니까? 그래프 순회로?
- 고객 → 주문 → 상품 → 프로모션
- 프로모션 → 제품 ← 반품 [correct]
- 반품 → 주문 → 고객 → 프로모션
- 프로모션 → 반품 → 상품
> Promotion → Product ← Return 경로는 PromotionForProduct 및 ReturnOfProduct 관계를 따르며 공유 Product 엔터티를 통해 프로모션을 반환된 제품에 연결합니다.
```
