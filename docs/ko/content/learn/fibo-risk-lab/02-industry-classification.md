## 업종을 분류하는 이유는 무엇인가요?

은행은 경제 부문 전반에 걸친 노출을 이해해야 합니다. 포트폴리오 대출의 40%가 건설회사에 대한 것이라면 주택 침체는 치명적일 수 있습니다. **NAICS**(북미 산업 분류 시스템)는 FIBO가 기반으로 하는 표준 분류 체계를 제공합니다.

이 단계에서는 위험 관련 속성이 풍부한 부문 → 하위 부문 → 산업 그룹의 3단계 계층 구조를 모델링합니다.

## 항목 유형

### 부문

가장 광범위한 분류는 "제조", "금융", "의료"를 생각해 보세요.

| 부동산 | 유형 | 메모 |
|---|---|---|
| `sectorCode` | 문자열 | 식별자(예: "31-33") |
| `sectorName` | 문자열 | 표시 이름 |
| `description` | 문자열 | 이 부문이 다루는 내용 |

### 하위 부문

부문 내의 하위 구분 - "제조" 내의 "식품 제조".

| 부동산 | 유형 | 메모 |
|---|---|---|
| `subsectorCode` | 문자열 | 식별자(예: "311") |
| `subsectorName` | 문자열 | 표시 이름 |

### 산업그룹

포트폴리오 분석에 중요한 위험 속성을 포함하는 가장 세부적인 수준입니다.

| 부동산 | 유형 | 메모 |
|---|---|---|
| `naicsCode` | 문자열 | 식별자 - 공식 NAICS 코드 |
| `name` | 문자열 | 업종명 |
| `cyclicality` | 문자열 | 경제 주기(예: '높음', '낮음', '경기대응')에 얼마나 민감한가요?
| `climateSensitivity` | 문자열 | 기후 현상에 대한 노출(예: "높음", "보통", "낮음") |
| `essentialServices` | 부울 | 업계가 필수 서비스를 제공하는지 여부(더 탄력적) |
| `description` | 문자열 | 산업 설명 |

## 관계

- **partOfSector**: `Subsector` → `Sector` (`many-to-one`) — 모든 하위 섹터는 정확히 하나의 섹터에 속합니다.
- **belongsToSubsector**: `IndustryGroup` → `Subsector` (`many-to-one`) — 모든 산업 그룹은 하위 부문에 속합니다.

이렇게 하면 엄격한 계층 구조가 생성됩니다. `Sector` ← `Subsector` ← `IndustryGroup`

## 디자인 패턴: 분류 계층

이는 가장 일반적인 온톨로지 패턴 중 하나입니다. 즉, 각 하위 항목이 정확히 하나의 상위 항목을 갖는 **엄격한 트리 계층 구조**입니다. 이를 통해 다음이 가능해집니다.

- **롤업 집계**: 하위 부문 노출을 얻기 위해 하위 부문 내의 IndustryGroups에 대한 모든 대출을 합산합니다.
- **세부 분석**: 부문 수준에서 시작하여 하위 부문을 심층 분석한 다음 특정 산업 그룹을 심층 분석합니다.
- **위험 속성 상속**: 섹터가 "순환"인 경우 모든 하위 섹터가 해당 위험 컨텍스트를 상속합니다.
