## 위험이 존재하는 곳

산업 분류는 귀하의 포트폴리오가 노출된 *어떤* 부문을 알려줍니다. 지리적 계층 구조는 *어디*를 알려줍니다. 플로리다에 집중된 포트폴리오는 캘리포니아에 집중된 포트폴리오와 허리케인, 지진, 산불 등 다양한 위험에 직면해 있습니다.

이 단계에서는 자연재해 플래그를 통해 더욱 세밀하게 지리적 위치를 모델링하는 엔터티 3개를 추가합니다.

## 새로운 항목 유형

### 지역

대륙 또는 거시 경제 지역 - "북미", "유럽", "아시아 태평양".

| 부동산 | 유형 | 메모 |
|---|---|---|
| `regionCode` | 문자열 | 식별자 |
| `regionName` | 문자열 | 표시 이름 |
| `description` | 문자열 | 지역 설명 |
| `disasterProfile` | 문자열 | 지역별 주요 재해 유형 |

### 국가

경제적, 규제적 속성을 지닌 민족국가.

| 부동산 | 유형 | 메모 |
|---|---|---|
| `countryCode` | 문자열 | 식별자(ISO 국가 코드) |
| `countryName` | 문자열 | 표시 이름 |
| `economicZone` | 문자열 | 경제적 분류(예: "선진", "신흥") |
| `currency` | 문자열 | 국가 통화 코드 |
| `regulatoryFramework` | 문자열 | 주요 은행 규제 기관 |

### 관할권

부울 재해 지역 플래그가 있는 하위 국가 관할권(주, 지방)입니다.

| 부동산 | 유형 | 메모 |
|---|---|---|
| `code` | 문자열 | 식별자(예: "FL", "CA") |
| `name` | 문자열 | 표시 이름 |
| `hurricaneZone` | 부울 | 허리케인 위험에 노출됨 |
| `floodZone` | 부울 | 홍수 위험에 노출됨 |
| `earthquakeZone` | 부울 | 지진 위험에 노출됨 |
| `wildfireZone` | 부울 | 산불 위험에 노출됨 |
| `coastal` | 부울 | 해안 관할권 |
| `latitude` | 십진수 | 지리적 위도 |
| `longitude` | 십진수 | 지리적 경도 |

## 새로운 관계

- **inCountry**: `Jurisdiction` → `Country` (`many-to-one`) — 각 관할권은 하나의 국가에 속합니다.
- **inRegion**: `Jurisdiction` → `Region` (`many-to-one`) — 각 관할권은 지리적 지역에 매핑됩니다.

## 디자인 패턴: 불리언 위험 플래그

Jurisdiction은 단일 "riskType" 열거형 대신 **부울 플래그**를 사용합니다. 이는 의도적인 것입니다. 관할권은 동시에 여러 재난 지역에 있을 수 있습니다. 플로리다는 `hurricaneZone`와 `floodZone`입니다. 캘리포니아는 `earthquakeZone`와 `wildfireZone`입니다.

이 패턴을 사용하면 정밀한 필터링이 가능합니다.

- "`hurricaneZone = true` 및 `coastal = true`인 모든 관할 구역 표시"
- "`earthquakeZone` 관할권에서 우리의 총 노출은 얼마입니까?"

## 두 개의 독립적인 계층

이 시점에서 모델에는 두 개의 별도 하위 그래프가 있습니다.

1. **산업**: 부문 ← 하위 부문 ← 산업 그룹
2. **지리**: 지역 ← 국가(관할권 경유) 및 지역 ← 관할권

이는 대출 상품 및 규제 한도를 추가할 때 이후 단계에서 연결됩니다.
