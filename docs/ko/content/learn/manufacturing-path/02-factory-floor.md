## IoT 기반

모든 스마트 팩토리는 두 가지 개념으로 시작됩니다.

- **기계** — 공장 현장에는 어떤 장비가 있나요?
- **센서** — 어떤 데이터를 생성합니까?

기계와 센서는 원격 측정 백본을 형성합니다. 생산이나 품질을 추적하기 전에 무엇이 실행되고 있고 무엇을 보고하는지 알아야 합니다.

## 엔터티 정의

### 기계

| 부동산 | 유형 | 식별자? |
|---|---|---|
| `machineId` | 문자열 | `name` |
| `type` | 문자열 | |
| `status` | 문자열 | |
| `installDate` | 문자열 | |
| ✓ | 날짜 | |

`status` 속성은 작동 상태(`running`, `idle`, `maintenance` 또는 `offline`)를 추적합니다. 이를 통해 실시간 대시보드 및 유지 관리 일정이 가능해집니다.

### 센서

| 부동산 | 유형 | 식별자? |
|---|---|---|
| `sensorId` | 문자열 | `type` |
| `unit` | 문자열 | |
| `lastReading` | 문자열 | |
| `threshold` | 플로트 | |
| ✓ | 플로트 | |

`threshold` 속성은 경고 경계를 정의합니다. `lastReading`가 `threshold`를 초과하면 시스템이 경보를 트리거합니다. 이 패턴은 예측 유지 관리의 기본입니다.

## 관계

- **모니터** — `Sensor` → `Machine`(다대일) 여러 센서가 동일한 기계를 모니터링합니다. 하나는 온도, 다른 하나는 진동 등입니다.

> **소유권 계층:** IoT 온톨로지에서 센서는 기계에 속합니다. 방향이 중요합니다. 센서는 기계를 모니터링하지만 그 반대 방향은 아닙니다. 이 상위-하위 계층 구조는 IoT 플랫폼이 원격 측정 데이터를 구성하는 방식입니다.

## 우리가 배운 것

- **IoT 계층**은 상위-하위 관계(센서 → 기계)를 사용합니다.
- **상태 속성**을 통해 실시간 운영 추적이 가능합니다.
- **임계값 속성**은 예측 유지 관리 경고를 지원합니다.
- 두 개의 엔터티라도 유용한 원격 측정 백본을 형성할 수 있습니다.

```quiz
Q: 센서 엔터티에 lastReading 및 임계값 속성이 모두 있는 이유는 무엇입니까?
- 잘못된 경우를 대비해 백업 값을 저장하기 위해
- 임계값은 경고 경계를 정의합니다. lastReading이 이를 초과하면 시스템은 예측 유지 관리에 대한 경보를 트리거합니다. [correct]
- 두 값 모두 모든 IoT 표준에서 요구됩니다.
- 임계값은 센서의 정확도를 계산하는 데 사용됩니다.
> 임계값 패턴은 예측 유지 관리의 기본입니다. 현재 판독값을 알려진 안전 경계와 비교함으로써 시스템은 자동으로 이상 현상을 감지하고 장비 고장이 발생하기 전에 운영자에게 경고할 수 있습니다.
```
