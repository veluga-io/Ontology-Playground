## 모니터링부터 프로듀싱까지

센서는 기계가 *어떻게* 작동하는지 알려주지만, 우리는 기계가 *무엇을* 생산하는지도 알아야 합니다. **작업 주문** 및 **부품** 엔터티는 공장 모델에 생산 추적을 추가합니다.

생산 추적을 추가하면 다음이 가능해집니다.
- "이번 교대조에서 가장 많은 부품을 생산하는 기계는 무엇입니까?"
- "일정보다 지연된 작업 주문은 몇 개입니까?"
- "CNC-01에서는 현재 어떤 부품이 생산되고 있나요?"

## 작업 주문 엔터티

| 부동산 | 유형 | 식별자? |
|---|---|---|
| `workOrderId` | 문자열 | `priority` |
| `status` | 문자열 | |
| `startDate` | 문자열 | |
| `dueDate` | 날짜 | |
| ✓ | 날짜 | |

작업 주문에는 `startDate` 및 `dueDate`가 모두 있으므로 일정 준수 계산이 가능합니다. `priority`와 결합하여 생산 계획 쿼리를 강화합니다.

## 부품 엔터티

| 부동산 | 유형 | 식별자? |
|---|---|---|
| `partId` | 문자열 | `name` |
| `material` | 문자열 | |
| `weight` | 문자열 | |
| `tolerance` | 플로트 | |
| ✓ | 플로트 | |

`tolerance` 속성은 허용 가능한 제조 편차를 정의합니다. 공차가 더 엄격한 부품에는 더 높은 정밀도의 기계가 필요합니다. 이는 주요 생산 계획 제약 사항입니다.

## 새로운 관계

- **할당_대상** — `Work-Order` → `Machine`(다대일) 작업 지시는 생산을 위해 특정 기계에 할당됩니다.

- **생산** — `Work-Order` → `Part`(일대다) 작업 주문은 하나 이상의 부품을 생산합니다.

- **has_part** — `Machine` → `Part`(일대다) 기계가 부품을 생산합니다(출력 관점).

> **생산 체인:** `Machine ← Work-Order → Part` 체인은 스케줄링 엔터티를 통해 장비를 출력에 연결합니다. 이는 약속이 의료 분야에서 환자와 제공자를 연결하는 방식과 유사합니다. 중간 엔터티는 이벤트를 나타냅니다.

## 우리가 배운 것

- **생산 체인**은 일정 엔터티(작업-주문)를 통해 장비를 출력에 연결합니다.
- **이중 날짜 속성**(startDate/dueDate)을 통해 일정 준수 추적 가능
- **공차 속성**은 제조 정밀도 요구 사항을 인코딩합니다.
- 이제 공장 모델에는 모니터링(센서)과 생산(작업 주문)이 모두 포함됩니다.

```quiz
Q: 부품 엔터티에서 공차 속성은 무엇을 나타냅니까?
- 결함이 있을 수 있는 최대 부품 수
- 허용 가능한 제조 편차 - 공차를 벗어나는 부품에는 고정밀 기계가 필요합니다. [correct]
- 부품 제작에 허용되는 시간
- 부품이 견딜 수 있는 온도 범위
> 공차는 부품의 실제 치수가 사양에서 벗어날 수 있는 정도를 정의합니다. 공차가 엄격해지면 더 높은 정밀도의 기계와 더 신중한 품질 관리가 필요하므로 이는 생산 계획의 주요 제약이 됩니다.
```
