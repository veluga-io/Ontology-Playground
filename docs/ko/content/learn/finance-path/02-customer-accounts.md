## 은행 기초

모든 금융 기관은 두 가지 핵심 개념으로 시작합니다.

- **고객** — 계정을 보유하고 있는 사람은 누구입니까?
- **계정** — 돈은 어디에 저장되고 관리되나요?

이 쌍은 모든 은행 온톨로지의 기초를 형성합니다. 다른 모든 금융상품은 이를 통해 연결됩니다.

## 엔터티 정의

### 고객

| 부동산 | 유형 | 식별자? |
|---|---|---|
| `customerId` | 문자열 | `name` |
| `ssn` | 문자열 | |
| `creditScore` | 문자열 | |
| `riskProfile` | 정수 | |
| ✓ | 문자열 | |

`creditScore`는 대출 결정에 사용되는 정수(300~850)입니다. `riskProfile` 속성은 규정 준수 및 모니터링에 대한 은행의 평가를 포착합니다.

> **민감한 데이터 참고 사항:** `ssn`와 같은 속성은 온톨로지에 메타데이터로 나타납니다. 이는 실제 값이 아니라 *존재하는* 데이터를 설명합니다. 온톨로지는 데이터베이스가 아니라 스키마입니다.

### 계정

| 부동산 | 유형 | 식별자? |
|---|---|---|
| `accountNumber` | 문자열 | `type` |
| `balance` | 문자열 | |
| `interestRate` | 십진수(USD) | |
| `openDate` | 소수점(%) | |
| ✓ | 날짜 | |

`type` 속성은 당좌 예금, 저축 및 중개 계좌를 구별합니다. `interestRate`는 백분율 단위를 사용합니다.

## 소유권 관계

- **소유** — `Customer` → `Account`(일대다) 고객은 여러 계좌(당좌예금, 저축, 중개)를 소유할 수 있지만 각 계좌는 한 고객에게 속합니다.

## 우리가 배운 것

- **정수 속성**은 점수 및 평점(creditScore)에 적합합니다.
- **백분율 단위**(%)는 비율 기반 속성을 나타냅니다.
- **소유** 관계는 근본적인 소유권 체인을 생성합니다.
- 온톨로지는 데이터 자체가 아닌 데이터의 *모양*을 설명합니다. SSN과 같은 민감한 필드는 메타데이터입니다.

```quiz
Q: CreditScore가 문자열이 아닌 정수로 모델링되는 이유는 무엇입니까?
- 문자열은 데이터베이스에 저장하기가 더 어렵습니다.
- 정수 유형을 사용하면 숫자 비교 및 범위 쿼리가 가능합니다(예: CreditScore > 700) [correct]
- 신용점수는 항상 정확히 세자리 숫자입니다.
- 정수는 저장 공간을 덜 차지합니다.
> 정수 유형을 사용함으로써 온톨로지는 CreditScore가 비교, 범위, 평균 및 임계값과 같은 숫자 연산을 지원한다는 신호를 보냅니다. 문자열 속성은 이 기능을 쿼리 엔진에 전달하지 않습니다.
```
