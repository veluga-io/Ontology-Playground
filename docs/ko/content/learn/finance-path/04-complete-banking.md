## 금융상품

기본 계좌 및 거래 외에도 은행은 두 가지 주요 상품 카테고리를 제공합니다.

- **대출** — 은행이 돈을 빌려주는 신용 상품
- **투자** — 고객이 부를 키우는 자산

이를 추가하면 그림이 완성되고 흥미로운 다중 경로 관계가 생성됩니다.

## 대출

| 부동산 | 유형 | 식별자? |
|---|---|---|
| `loanId` | 문자열 | `principal` |
| `apr` | 십진수(USD) | |
| `term` | 소수점(%) | |
| `status` | 정수(월) | |
| ✓ | 문자열 | |

`term`는 기간 속성의 일반적인 패턴인 월 단위로 측정된 정수입니다. `apr`(연간 백분율 비율)는 백분율 단위를 사용합니다.

## 투자

| 부동산 | 유형 | 식별자? |
|---|---|---|
| `holdingId` | 문자열 | `symbol` |
| `shares` | 문자열 | |
| `purchasePrice` | 십진수 | |
| `currentValue` | 십진수(USD) | |
| ✓ | 십진수(USD) | |

`symbol` 속성(예: MSFT, AAPL)은 주식을 식별합니다. `purchasePrice`와 `currentValue`를 모두 사용하면 이익/손실 계산이 가능합니다.

## 새로운 관계

금융 상품을 연결하는 네 가지 관계는 다음과 같습니다.

- **has_loan** — `Customer` → `Loan`(일대다) 고객은 여러 대출을 받을 수 있습니다.

- **자금** — `Account` → `Loan`(일대다) 계정은 대출금 상환을 위한 지불 소스 역할을 합니다.

- **보류** — `Customer` → `Investment`(일대다) 고객의 투자 포트폴리오입니다.

- **linked_to** — `Account` → `Investment` (일대다) 투자 보유 자산과 연결된 중개 계좌입니다.

> **다중 경로 패턴:** 투자는 *두 가지* 다른 경로(직접 `holds`를 통해, 간접적으로 `Account → linked_to`를 통해)를 통해 고객에게 연결됩니다. 이러한 중복성은 의도적인 것입니다. 이는 소유권(누가 보유하고 있습니까?)과 자금(어떤 계정이 이를 뒷받침합니까?)을 모두 모델링합니다.

*완전한 은행 및 금융 온톨로지: 5개 엔터티, 6개 관계. 대출과 투자는 고객과 계좌를 통해 연결됩니다.*

## 완전한 모델이 가능하게 하는 것

| 질문 | 그래프 경로 |
|---|---|
| 고위험 고객 중 고액 대출을 받은 고객은 누구입니까? | 고객(riskProfile=높음) → 대출(원금 > 100K) |
| 상위 고객의 포트폴리오 가치는 무엇입니까? | 고객 → 투자(현재 가치 합계) |
| 대출과 투자 모두에 자금을 지원하는 계좌는 무엇입니까? | 계정 → 대출 AND 계정 → 투자 |
| 어떤 고객의 투자가 대출 비용을 능가합니까? | 고객 → 투자(현재가치) vs 고객 → 대출(원금×4월) |

## GQL 쿼리 예

투자 포트폴리오가 총 대출 원금을 초과하는 고객을 찾으십시오.

```gql
MATCH (c:Customer)-[:holds]->(inv:Investment),
      (c)-[:has_loan]->(loan:Loan)
WITH c, SUM(inv.currentValue) AS portfolio, SUM(loan.principal) AS debt
WHERE portfolio > debt
RETURN c.name, portfolio, debt
```

## 우리가 만든 것

| 단계 | 추가된 엔터티 | 누적 | 핵심 개념 |
|---|---|---|---|
| 1 | 고객, 계정 | 2 | 소유권, 금융 식별자 |
| 2 | 거래 | 3 | 활동 추적, 날짜/시간 정밀도 |
| 3 | 대출, 투자 | 5 | 금융 상품, 다중 경로 관계 |

## 주요 내용

1. **소유권 체인**(고객 → 계정 → 거래)을 통해 규정 준수 쿼리가 가능합니다.
2. **날짜/시간 정밀도**는 재무 데이터에 매우 중요합니다.
3. **다중 경로 관계**는 동일한 연결의 다양한 측면을 모델링합니다.
4. **기간 속성**(월 단위)은 단위가 있는 정수를 사용합니다.
5. 금융 온톨로지는 데이터 형태를 설명합니다. 민감한 데이터는 소스 시스템에 유지됩니다.

```quiz
Q: 투자가 고객("holds"를 통해)과 계좌("linked_to"를 통해) 모두에 연결되는 이유는 무엇입니까?
- 실수입니다. 단 하나의 관계만 있으면 됩니다.
- 각 관계는 소유권과 자금 출처 등 서로 다른 측면을 모델링합니다. [correct]
- 투자가 유효하려면 최소한 두 가지 관계가 필요합니다.
- 일대다 관계는 항상 쌍으로 나타납니다.
> "보유" 관계는 "이 투자의 소유자는 누구입니까?"라고 대답합니다. "linked_to"는 "어느 계정에서 자금을 조달합니까?"라고 대답합니다. 이는 답변이 다를 수 있는 다양한 질문입니다(예: 한 사람의 투자에 자금을 지원하는 공동 계좌).
```
