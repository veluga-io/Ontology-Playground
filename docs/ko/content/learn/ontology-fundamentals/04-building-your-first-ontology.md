## 우리가 만들 것

이 튜토리얼에서는 관계로 연결된 `Book`, `Author` 및 `Member`의 세 가지 엔터티 유형을 사용하여 간단한 **라이브러리** 온톨로지를 만듭니다. 마지막에는 Microsoft Fabric IQ 또는 의미 체계 도구와 함께 사용할 수 있는 유효한 RDF 파일이 준비됩니다.

## 1단계: 디자이너 열기

상단 탐색 모음에서 **디자이너** 버튼을 클릭하거나 [/#/designer](#/designer)로 직접 이동하세요. 빈 캔버스가 표시됩니다. 왼쪽에는 엔터티 양식이 있고 오른쪽에는 실시간 그래프 미리 보기가 있습니다.

## 2단계: 항목 유형 만들기

**+ 엔터티 추가** 버튼을 사용하여 세 개의 엔터티를 추가합니다.

**예약**
- 이름 : `Book`
- 아이콘: `📚`
- 색상 : 파란색을 선택하세요
- 속성:
  - `isbn` — 문자열, **식별자** `title`
  - `publishedYear` — 문자열
  - ✓ — 정수

**저자**
- 이름 : `Author`
- 아이콘: `✍️`
- 색상 : 녹색을 선택하세요
- 속성:
  - `authorId` — 문자열, **식별자** `name`
  - `nationality` — 문자열
  - ✓ — 문자열

**회원**
- 이름 : `Member`
- 아이콘: `👤`
- 색상 : 보라색을 선택하세요
- 속성:
  - `memberId` — 문자열, **식별자** `name`
  - `joinDate` — 문자열
  - ✓ — 날짜

각 엔터티를 추가할 때 실시간으로 업데이트되는 그래프 미리보기를 확인하세요.

## 3단계: 관계 추가

**관계** 탭으로 전환하고 다음을 추가합니다.

| 관계 | 보낸 사람 | 에 | 카디널리티 |
|-------------|------|------|-------------|
| `writtenBy` | 예약 | 작성자 | 다대일 |
| `borrowedBy` | 예약 | 회원 | 다대다 |

`writtenBy` 관계는 많은 책이 한 명의 저자를 공유할 수 있지만 각 책에는 한 명의 주 저자가 있으므로 다대일 관계입니다. `borrowedBy` 관계는 한 권의 책을 여러 회원이 빌릴 수 있고, 한 회원이 여러 권의 책을 빌릴 수 있으므로 다대다 관계입니다.

## 4단계: 유효성 검사

툴바에서 **확인** 버튼을 클릭하세요. 모든 것이 정확하면 녹색의 "문제 없음" 배너가 표시됩니다. 그렇지 않은 경우 보고된 문제를 수정하세요.

- 모든 엔터티에는 적어도 하나의 식별자 속성이 있어야 합니다.
- 관계는 기존 엔터티 유형을 참조해야 합니다.
- 중복된 ID가 없습니다.

## 5단계: RDF 미리보기

미리보기 창에서 **Fabric IQ** 탭을 클릭하세요. 구문 강조가 포함된 라이브 RDF/OWL 출력을 볼 수 있습니다. 이것은 RDF와 같은 도구가 사용하는 정확한 파일입니다.

<ontology-embed id="official/cosmic-coffee" height="400px"></ontology-embed>

*Fourth Coffee 온톨로지는 동일한 작업 흐름을 사용하여 구축되었습니다. 라이브러리 온톨로지는 유사하게 보일 것입니다. 엔터티는 다채로운 노드로, 관계는 방향이 지정된 가장자리로 표시됩니다.*

## 6단계: 내보내기

세 가지 옵션이 있습니다:

1. **`.rdf` 다운로드** — GitHub 파일을 다운로드 폴더에 저장합니다.
2. **카탈로그에 제출** — 원클릭 RDF 흐름을 열어 온톨로지를 커뮤니티 카탈로그에 기여합니다(JSON 로그인 필요)
3. **JSON 복사** — 앱에서 사용하기 위해 PR 표현을 복사합니다.

## 다음은 무엇입니까?

- [카탈로그](#/catalogue)를 탐색하여 다른 온톨로지가 어떻게 구성되어 있는지 확인하세요.
- 명명 규칙 및 모범 사례는 [온톨로지 디자인 패턴](#/learn/ontology-design-patterns)을 참조하세요.
- 홈페이지의 **Query Playground**를 통해 온톨로지에 대해 자연어 질문을 해보세요.

## 주요 내용

- 디자이너는 온톨로지 구축을 위한 시각적이고 코드가 없는 작업 흐름을 제공합니다.
- 모든 엔터티에는 이름, 하나 이상의 속성 및 하나의 식별자가 필요합니다.
- 관계는 엔터티를 이름과 카디널리티로 연결합니다.
- 실시간 그래프와 Fabric IQ 미리보기는 디자인과 동시에 즉각적인 피드백을 제공합니다.
- RDF를 위해 RDF로 내보내거나 커뮤니티 카탈로그에 직접 제출

```quiz
Q: Book과 Member 사이의 BorrowedBy 관계가 다대다로 설정된 이유는 무엇입니까?
- 책은 한 번만 빌릴 수 있습니다.
- 각 회원은 한 번에 정확히 한 권의 책을 빌립니다.
- 한 권의 책은 시간이 지남에 따라 여러 회원이 빌릴 수 있으며, 회원은 여러 권의 책을 빌릴 수 있습니다. [correct]
- 다대다 방식은 모든 관계의 기본 카디널리티입니다.
> 한 권의 책을 여러 회원이 서로 다른 시간에 빌릴 수 있고, 각 회원이 여러 권의 책을 동시에 빌릴 수 있습니다. 이러한 양방향 다중성이 다대다를 가능하게 합니다.
```
