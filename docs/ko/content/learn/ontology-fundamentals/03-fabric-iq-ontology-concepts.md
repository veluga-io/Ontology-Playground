## Fabric IQ란 무엇인가요?

**Microsoft Fabric**는 데이터 엔지니어링, 데이터 과학, 실시간 분석 및 비즈니스 인텔리전스를 통합한 통합 분석 플랫폼입니다. **IQ**는 사용자가 **자연어**로 질문하고 구조화된 데이터에서 답변을 얻을 수 있는 Fabric 기능입니다. SQL이 필요하지 않습니다.

핵심 요소는 **온톨로지**입니다. 즉, 엔터티 유형, 해당 속성 및 관계에 대한 공식적인 설명입니다. IQ는 온톨로지를 읽고 데이터의 형태를 이해하며 일반 영어 질문을 올바른 쿼리로 번역합니다.

## IQ가 온톨로지를 사용하는 방법

사용자가 "지난달 지역별 총 매출은 얼마입니까?"*라고 묻는 경우 IQ는 다음 사항을 알아야 합니다.

1. **엔티티 유형** — `Order`, `Store`, `Region`
2. **속성** — `Order.totalAmount`, `Order.date`, `Store.region`
3. **관계** — `Order` → `placedAt` → `Store`, `Store` → `locatedIn` → `Region`
4. **식별자 속성** — 각 엔터티를 고유하게 식별하는 필드(예: `Order.orderId`)

온톨로지는 네 가지를 모두 제공합니다. 그것이 없으면 IQ는 "상점"과 "제품"을 구별할 수 없으며 이에 가입하는 방법도 알 수 없습니다.

## 항목 유형

엔터티 유형은 비즈니스 개체의 범주입니다. Fabric IQ에서 각 엔터티 유형은 다음과 같습니다.

- **이름**과 선택적 **설명**이 있습니다.
- 하나 이상의 **속성**(입력된 열)을 포함합니다.
- 인스턴스를 고유하게 식별하는 **식별자 속성**이 하나 이상 있어야 합니다.

테이블 정의(`Customer(customerId, name, email, tier)`)로 생각하세요.

## 속성 및 유형

각 속성에는 다음과 같은 데이터 유형이 있습니다.

| 유형 | 설명 | 예 |
|------|-------------|---------|
| `string` | 텍스트 값 | 고객 이름, 제품 SKU |
| `integer` | 정수 | 수량, 연도 |
| `decimal` | 분수 | 가격, 평가 |
| `date` | 달력 날짜 | 주문일, 생년월일 |
| `datetime` | 날짜와 시간 | 타임스탬프가 생성됨 |
| `boolean` | 참/거짓 | 활성, 프리미엄 |

**식별자 속성**(열쇠 아이콘으로 표시)은 매우 중요합니다. 이는 IQ에게 엔터티를 올바르게 계산하고 그룹화하고 조인하는 방법을 알려줍니다.

## 관계 및 카디널리티

관계는 엔터티 유형을 연결합니다. 각 관계는 다음을 지정합니다.

- **소스 및 대상** 항목 유형
- **이름**(동사: "places", "contains", "worksAt")
- **카디널리티** — 연결할 수 있는 인스턴스 수

| 카디널리티 | 의미 | 예 |
|------------|---------|---------|
| 일대일 | 각 A는 정확히 하나의 B | `Employee` → `Badge` |
| 일대다 | 각 A는 여러 B에 매핑됩니다 | `Customer` → `Order` |
| 다대일 | 많은 As가 하나의 B에 매핑됨 | `Order` → `Store` |
| 다대다 | 많은 B에 대한 많은 As 매핑 | `Student` → `Course` |

IQ는 카디널리티를 사용하여 올바른 집계를 생성합니다. `Customer`와 `Order` 간의 일대다 관계는 "고객당 주문 수"가 유효한 반면 "주문당 고객 수"는 일반적으로 1임을 의미합니다.

## IQ를 위한 디자인

Fabric IQ에 대한 온톨로지를 구축할 때 다음 지침을 따르십시오.

1. **엔터티 이름을 명확하게 지정** — 사용자가 말할 비즈니스 용어를 사용합니다("tbl_cust"가 아닌 "고객").
2. **설명 추가** — IQ는 유사한 개념을 명확하게 하기 위해 설명을 사용합니다.
3. **식별자 표시** — 모든 엔터티에는 최소한 하나의 식별자 속성이 있어야 합니다.
4. **카디널리티 설정** — IQ가 올바른 GROUP BY 및 JOIN 논리를 생성하는 데 도움이 됩니다.
5. **집중 유지** — 모든 내부 테이블이 아니라 사용자가 쿼리할 개념을 모델링합니다.

## 주요 내용

- IQ는 온톨로지를 사용하여 자연어 질문을 SQL로 변환합니다.
- 엔터티 유형, 속성, 관계 및 카디널리티가 네 가지 기본 요소입니다.
- 모든 엔터티에는 올바른 계산 및 결합을 위한 식별자 속성이 필요합니다.
- 좋은 이름과 설명은 IQ의 질문 답변 정확도를 향상시킵니다.
- IQ용 온톨로지를 시각적으로 만들려면 [온톨로지 디자이너](#/designer)를 사용하세요.
- [온톨로지 디자이너](Fabric IQ)를 사용하여 IQ 지원 온톨로지를 시각적으로 생성

```quiz
Q: Fabric IQ의 모든 엔터티 유형에 식별자 속성이 필요한 이유는 무엇입니까?
- 온톨로지를 전문적으로 보이게 만듭니다.
- IQ에게 엔터티를 올바르게 계산, 그룹화 및 조인하는 방법을 알려줍니다. [correct]
- 엔터티의 표시 이름으로 사용됩니다.
- 기본 정렬 순서를 설정합니다.
> 식별자 속성은 엔터티 유형의 인스턴스를 고유하게 구별합니다. 이것이 없으면 IQ는 변환된 SQL에서 COUNT, GROUP BY 또는 JOIN 작업을 올바르게 생성할 수 없습니다.
```

```quiz
Q: 관계의 카디널리티는 Fabric IQ에 무엇을 말합니까?
- 관계를 그릴 때 사용할 색상
- 관계의 각 측면에 연결할 수 있는 인스턴스 수 [correct]
- 관계가 선택사항인지 필수인지 여부
- 엔터티가 표시되어야 하는 순서
> 카디널리티(일대일, 일대다, 다대일, 다대다)는 IQ에 올바른 집계 및 조인을 생성하는 방법을 알려줍니다. 예를 들어 한 고객이 많은 주문을 받았다는 것을 알 수 있습니다.
```
