## 교수진 추가하기

누가 강의를 가르칩니까? **교수** 엔터티는 교육 차원을 추가하여 교수진과 강좌, 그리고 간접적으로 학생들과 연결합니다.

교수님을 추가하면 다음과 같은 기능을 사용할 수 있습니다.
- "어떤 교수가 가장 많은 400급 과정을 가르치는가?"
- "스미스 교수의 강의에서 평균 GPA는 얼마입니까?"
- "어떤 정임 교수들이 입문 과정을 가르치나요?"

## 교수 엔터티

| 재산 | 유형 | 식별자? |
|---|---|---|
| `professorId` | 문자열 | ✓ |
| `name` | 문자열 | |
| `rank` | 문자열 | |
| `tenured` | 논리형 | |
| `officeHours` | 문자열 | |

`rank` 속성(조수, 부원, 전임)은 학문적 계층을 반영합니다. `tenured` 부울 값은 직업 안정성과 기관 투자에 대한 질의를 가능하게 합니다.

## 새로운 관계들

- **강사** - `Professor` → `Course` (일대다수) 교수는 학기에 한두 개 이상의 과정을 가르칩니다.

- **상담** — `Professor` → `Student` (일대다) 교수는 학생들의 학업 프로그램에 대해 조언합니다.

> **전속적인 쿼리:** 교수 → 과정 ← 등록 ← 학생으로, 이제 교수-과정-등록-학생의 관계를 가로지르는 질문을 할 수 있습니다. 예를 들어, "임용 교수로부터 과정을 수강하는 학생은 누구입니까?" 이는 교수 → 과정 → 등록 → 학생을 순회하는 것을 필요로 합니다.

## 우리가 배운 것

- **보울륨 프로퍼티**(임기제)는 필터링에 대한 예/아니오 분류를 생성합니다.
- **전속적 쿼리** 먼 엔터티를 연결하기 위해 여러 관계를 탐색합니다.
- **학술 순위**는 정의된 계층 구조에 따라 순위가 정해져 있습니다(조교수 → 조교 → 전문교수)
- 이 그래프는 이제 학생 중심과 교수님 중심의 쿼리를 모두 지원합니다.

```quiz
Q: 대학 오노토피아를 가로질러의 순차적 쿼리는 어떻게 나타나요?
- 단일 엔터티의 속성을 조회하기
- 교수 → 과정 → 등록 → 학생과 같은 여러 관계를 가로질러 먼 엔터티를 연결하는 것 [correct]
- 교수를 그들의 ID로 찾아보기
- 시스템 내 과정의 수를 세기
> 순차적 쿼리는 그래프 기반 엔톨로지 중 가장 큰 장점 중 하나입니다. 교수 → 과정 → 등록 → 학생을 순회함으로써, "어떤 학생들이 정년 교수의 수업에 있습니까?"와 같은 질문에 답할 수 있습니다. 이는 직접적인 관계가 없지만 중간 노드를 통해 연결된 엔터티를 연결하는 것입니다.
```

다음으로, 학문적 구조를 정리하기 위해 학부를 추가하겠습니다.
