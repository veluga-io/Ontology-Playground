## 조직 구조

대학은 **학과**로 조직됩니다. 학과는 교수진을 수용하고, 과정을 제공하고, 학위를 수여하는 행정 단위입니다. 학과를 추가하면 모든 것을 연결하는 조직 계층 구조를 만듭니다.

## 부서 엔터티

| 재산 | 유형 | 식별자? |
|---|---|---|
| `departmentId` | 문자열 | ✓ |
| `name` | 문자열 | |
| `building` | 문자열 | |
| `budget` | 플로트 | |
| `headOfDept` | 문자열 | |

`budget` 플로트는 자원 할당 쿼리를 가능하게 합니다. `headOfDept` 속성은 학과를 이끄는 교수를 참조합니다. 이는 조직 계층 구조에서 흔히 볼 수 있는 자기 참조 패턴입니다.

## 새로운 관계들

- **belongs_to** — `Professor` → `Department` (여러개에서 하나로) 교수는 학과에 소속되어 있습니다.

- **제공** — `Department` → `Course` (일대다) 학과는 학업 프로그램의 일환으로 과목을 제공합니다.

> **조직 계층:** 학과는 대학 오노토지에 있는 맨 위에 위치합니다. 그것은 교수(교직원)와 과정(교과) 모두와 하향으로 연결됩니다. 이 허브 위치는 학부를 집계 쿼리인 "학과 수준 통계"에 이상적으로 만듭니다.

## 전체 그래프

*전체 대학 온톨로지: 5개 엔터티, 6개 관계. 학과는 교수진과 커리큘럼 모두를 조직합니다.*

## 완전한 모델이 무엇을 가능하게 하는가

| 질문 | 그래프 경로 |
|---|---|
| 어떤 학과가 학생 평균 GPA가 가장 높은가요? | 학과 → 과정 ← 등록 → 학생(평균 GPA) |
| 어떤 교수들은 자신의 학과의 수업 이외에도 수업을 가르칩니까? | 교수 → 학과 vs 교수 → 수업 → 학과 |
| 각 학과의 등록률은 얼마입니까? | 학과 → 과정 ← 등록(개수) / 과정.maxEnrollment |
| 어떤 학과가 가장 오랜 기간 재직 중인 교수진을 보유하고 있나요? | 학과 ← 교수(재직=true, 수량) |

## GQL 쿼리 예시

학생들이 어려움을 겪고 있는 부서(평균 학점이 B 이하인 부서)를 찾으십시오.

```gql
MATCH (d:Department)-[:offers]->(c:Course)<-[:for_course]-(e:Enrollment)<-[:enrolls_in]-(s:Student)
WHERE e.grade IN ['C', 'D', 'F']
RETURN d.name, c.title, COUNT(e) AS struggling_count
ORDER BY struggling_count DESC
```

## 우리가 만든 것

| 단계 | 추가된 엔터티 | 누적 | 핵심 개념 |
|---|---|---|---|
| 1 | 학생, 과정, 등록 | 3 | 교차 엔터티, 다대다 |
| 2 | 교수 | 4 | 순차적 쿼리, 논리적 속성 |
| 3 | 부서 | 5 | 조직 계층 구조, 허브 엔터티 |

## 주요 내용 요약

1. **접합 엔터티**(등록)는 속성과 관련된 다대다 관계를 해결합니다.
2. **전위 쿼리** 다중 홉 경로를 통해 통찰력을 잠금 해제합니다.
3. **보울륨 프로퍼티**(임기제)는 범주 필터링을 가능하게 합니다.
4. **조직 계층**(부서)은 집계 그룹화를 제공합니다.
5. **허브 엔터티**(부서)는 온톨로지의 여러 지부를 연결합니다.

```quiz
Q: 왜 학과는 대학 오노토지에 "허브 엔터티"로 간주됩니까?
- 왜냐하면 그것은 가장 많은 속성을 가지고 있기 때문이다
- 조직 계층 구조의 맨 위에 앉아 있는 교수님과 과정과 모두 연결되기 때문에 [correct]
- 마지막에 추가되었기 때문에
- 허브 엔터티에는 예산 속성이 있어야 하기 때문에
> 학과는 교수님(belongs_to를 통해)과 과정(offers를 통해) 모두와 하향 연결됩니다. 이 이중 연결은 학과 수준에서 교수진과 커리큘럼 데이터를 결합하는 집계 쿼리에 이상적입니다.
```
