## 사진 완성

Fourth Coffee는 커피만 판매하는 것이 아닙니다. 전 세계 공급업체로부터 원두를 공급받고, 매장에서 배송을 받고, 전체 공급망을 추적합니다. **공급업체** 및 **배송**을 추가하면 루프가 닫힙니다.

## 공급업체

| 부동산 | 유형 | 식별자? |
|---|---|---|
| `supplierId` | 문자열 | `name` |
| `country` | 문자열 | |
| `certification` | 문자열 | |
| `rating` | enum (공정 무역, 열대우림 동맹, 유기농, 직접 거래, 없음) | |
| ✓ | 십진수 | |

`certification` 속성은 지속 가능성 자격 증명을 캡처하는 열거형입니다. `rating`는 품질 점수를 나타내는 소수점(1~5)입니다.

## 배송

| 부동산 | 유형 | 식별자? |
|---|---|---|
| `shipmentId` | 문자열 | `dispatchDate` |
| `arrivalDate` | 날짜 | |
| `status` | 날짜 | |
| `weight` | enum(배송 중, 배달됨, 지연됨) | |
| ✓ | 십진수(kg) | |

배송은 **허브 엔터티** 역할을 합니다. 이는 제품을 통해 공급업체를 매장에 연결하고 비즈니스의 소싱 및 소매 측면을 연결합니다.

## 새로운 관계

네 가지 새로운 관계가 공급망을 완성합니다.

- **sourcedFrom** — `Product` → `Supplier` (다대일) 각 제품의 원두는 하나의 공급자로부터 공급됩니다.

- **sentBy** — `Shipment` → `Supplier`(다대일) 각 배송은 하나의 공급업체에서 시작됩니다.

- **deliveredTo** — `Shipment` → `Store` (다대일) 각 배송물은 하나의 매장에 도착합니다.

- **운송** — `Shipment` → `Product`(다대다) 한 배송에 여러 제품이 포함될 수 있으며, 한 제품이 여러 배송에 포함될 수 있습니다.

> **허브 엔터티 패턴:** 배송은 세 가지 다른 엔터티(공급업체, 매장, 제품)를 연결합니다. 허브 엔터티는 그래프에서 연결이 끊어진 부분을 연결하므로 강력합니다.

## 전체 그래프

*완전한 Fourth Coffee 온톨로지: 6개 엔터티 유형, 7개 관계. 배송은 공급업체, 매장, 제품을 연결하는 허브 역할을 합니다.*

## 완전한 모델이 가능하게 하는 것

| 질문 | 그래프 경로 |
|---|---|
| 유기농 콩을 공급하는 공급업체는 어디입니까? | 제품(isOrganic=true) → 공급업체 |
| 배송이 지연된 매장은 어디인가요? | 배송(상태=지연) → 매장 |
| 우리의 최고 공급업체의 등급은 어떻습니까? | 제품 → 공급업체(등급별 정렬) |
| 어느 인증 공급업체가 우리의 가장 큰 매장에 제품을 배송합니까? | 공급자 → 출하 → 매장(용량별 정렬) |

## GQL 쿼리 예

캘리포니아 내 매장으로 배송하는 공정 무역 인증을 받은 공급업체를 찾으세요.

```gql
MATCH (sup:Supplier)<-[:sentBy]-(s:Shipment)-[:deliveredTo]->(st:Store)
WHERE sup.certification = 'Fair Trade' AND st.state = 'CA'
RETURN sup.name, st.name, s.status
```

## 우리가 만든 것

| 단계 | 추가된 엔터티 | 누적 | 핵심 개념 |
|---|---|---|---|
| 1 | 고객, 주문, 상품 | 3 | 엔터티 유형, 식별자, 카디널리티 |
| 2 | 매장 | 4 | 위치 모델링, 다대일 |
| 3 | 공급자, 선적 | 6 | 공급망, 허브 엔터티 |

## 주요 내용

1. **작게 시작** - 세 가지 엔터티만으로도 가치를 창출할 수 있습니다.
2. 배송 브리지와 같은 다양한 비즈니스 도메인과 같은 **허브 엔터티**
3. **Enum 속성**은 모델 수준에서 데이터 품질을 강화합니다.
4. **그래프가 점진적으로 커집니다** - 각 단계마다 새로운 쿼리 기능이 추가됩니다.
5. **GQL 쿼리**는 온톨로지 구조에 직접 매핑되며 임피던스 불일치가 없습니다.

```quiz
Q: 이 온톨로지에서 배송이 "허브 엔터티"로 간주되는 이유는 무엇입니까?
- 어떤 개체보다 가장 많은 속성을 가지고 있습니다.
- 공급업체, 매장, 제품의 세 가지 다른 엔터티를 연결합니다. [correct]
- 가장 많이 문의하는 항목입니다.
- 모델에 마지막으로 추가된 엔터티입니다.
> 배송은 공급업체(sentBy), 매장(deliveredTo) 및 제품(운송)과 관계가 있어 소싱, 물류 및 소매 도메인을 단일 엔터티로 연결하므로 허브입니다.
```
