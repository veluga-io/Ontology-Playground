## 구매 전

모든 탐색 세션이 구매로 이어지는 것은 아닙니다. **장바구니**는 구매자가 결제하기 전에 고려하고 있는 사항을 포착합니다. 임시적이고 변경 가능한 세션 엔터티입니다.

카트를 추가하면 다음과 같은 질문에 답할 수 있습니다.
- "이번 주에 버려진 카트는 몇 개나 됩니까?"
- "평균 장바구니 금액과 평균 주문 금액은 얼마인가요?"
- "장바구니에 가장 자주 추가되지만 구매되지 않는 제품은 무엇입니까?"

## 장바구니 엔터티

| 부동산 | 유형 | 식별자? |
|---|---|---|
| `cartId` | 문자열 | `createdAt` |
| `itemCount` | 날짜/시간 | |
| `subtotal` | 정수 | |
| ✓ | 십진수(USD) | |

`itemCount` 및 `subtotal`는 비정규화된 요약 속성입니다. 카트 콘텐츠에서 계산할 수 있지만 직접 저장하면 쿼리가 더 빨라집니다.

## 새로운 관계

- **has_cart** — `Buyer` → `Shopping-Cart`(일대일) 각 구매자는 정확히 하나의 활성 카트를 가지며 각 카트는 정확히 한 명의 구매자에게 속합니다.

- **포함** — `Shopping-Cart` → `Product`(다대다) 카트에는 여러 제품이 포함될 수 있으며 제품은 여러 카트에 포함될 수 있습니다.

> **일대일 패턴:** 각 구매자가 단일 활성 쇼핑 세션을 갖기 때문에 `has_cart` 관계는 일대일입니다. 이는 구매자가 시간이 지남에 따라 주문을 누적하지만 언제든지 장바구니가 하나만 있다는 점에서 주문(일대다)과 다릅니다.

*쇼핑 카트는 두 가지 새로운 관계를 통해 구매자를 제품에 연결합니다. 차이점은 1단계 이후 변경된 사항을 강조합니다.*

## 우리가 배운 것

- **세션 엔터티**는 임시 또는 진행 중 상태(카트, 초안, 세션)를 모델링합니다.
- **일대일 관계** 엄격한 페어링(구매자 1명 ← 장바구니 1개)을 시행합니다.
- **비정규화된 속성**(itemCount, 소계) 쿼리 속도를 위한 저장 공간 교환
- 장바구니 분석을 통해 **전환 유입경로** 인사이트 제공(장바구니 → 주문 비율)

```quiz
Q: 구매자와 Shopping-Cart 간의 has_cart 관계가 일대다 대신 일대일로 설정되는 이유는 무엇입니까?
- 장바구니에는 고유 식별자가 필요하지 않기 때문입니다.
- 각 구매자는 특정 시간에 정확히 하나의 활성 카트를 갖고 있기 때문에 [correct]
- 일대일로 구현하는 것이 더 간단하기 때문입니다.
- 구매 후 장바구니가 삭제되기 때문에
> 구매자는 한 번에 하나의 활성 쇼핑 세션(장바구니)을 유지합니다. 구매자의 평생 동안 누적되는 주문과 달리 카트는 구매자 한 명, 활성 카트 한 명으로 구성된 현재 상태의 개체입니다.
```
