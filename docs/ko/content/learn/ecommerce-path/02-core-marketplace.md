## 구매 흐름

모든 마켓플레이스는 세 가지 개념을 중심으로 진행됩니다.

- **구매자** — 누가 구매하나요?
- **제품** — 판매되는 제품은 무엇입니까?
- **주문** — 완료된 거래는 무엇이었나요?

이 세 가지 엔터티는 필수 구매 흐름을 포착합니다. 우리가 추가하는 모든 것들은 이 기반을 더욱 풍요롭게 합니다.

## 엔터티 정의

### 구매자

| 부동산 | 유형 | 식별자? |
|---|---|---|
| `buyerId` | 문자열 | `email` |
| `memberSince` | 문자열 | |
| `loyaltyTier` | 날짜 | |
| `totalSpent` | 문자열 | |
| ✓ | 십진수(USD) | |

실제 소매 고객과 달리 전자 상거래 구매자는 항상 기본 연락 방법으로 `email`를 사용합니다. `totalSpent` 속성은 평생 가치 세분화를 활성화합니다.

### 제품

| 부동산 | 유형 | 식별자? |
|---|---|---|
| `sku` | 문자열 | `name` |
| `category` | 문자열 | |
| `price` | 문자열 | |
| `stockQty` | 십진수(USD) | |
| ✓ | 정수 | |

여기서 식별자는 전자상거래의 표준 제품 식별자인 `sku`(재고 유지 단위)입니다. `stockQty` 속성은 실시간 재고를 추적합니다.

### 주문

| 부동산 | 유형 | 식별자? |
|---|---|---|
| `orderId` | 문자열 | `orderDate` |
| `status` | 날짜/시간 | |
| `total` | 문자열 | |
| `shippingMethod` | 십진수(USD) | |
| ✓ | 문자열 | |

## 관계

- **장소** — `Buyer` → `Order`(일대다) 구매자는 시간이 지남에 따라 여러 주문을 할 수 있습니다.

- **포함** — `Order` → `Product`(다대다) 주문에는 여러 제품이 포함될 수 있으며 각 제품은 여러 주문에 걸쳐 나타납니다.

## 우리가 배운 것

- **SKU**는 전자상거래 제품의 표준 식별자입니다.
- `stockQty` 정수 속성을 사용하면 인벤토리 쿼리가 가능합니다.
- 기본 구매 흐름(구매자 → 주문 → 제품)은 모든 마켓플레이스의 중추입니다.

```quiz
Q: "productId" 대신 "sku"가 제품 식별자로 사용되는 이유는 무엇입니까?
- SKU는 유형이 더 짧습니다.
- SKU(Stock Keeping Unit)는 전자상거래 및 소매 시스템의 표준 제품 식별자입니다. [correct]
- productId로 인해 이름 지정 충돌이 발생합니다.
- SKU는 항상 숫자 값입니다.
> SKU는 Stock Keeping Unit의 약어입니다. 이는 재고 관리, 창고 보관 및 전자 상거래 시스템 전반에서 각 품목을 고유하게 식별하는 데 사용되는 업계 표준 식별자입니다.
```
