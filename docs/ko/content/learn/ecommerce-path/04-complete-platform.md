## 피드백 루프 닫기

전자상거래 퍼즐의 마지막 조각은 **고객 리뷰**입니다. 리뷰는 구매자를 제품과 다시 연결하여 향후 구매에 영향을 미치는 피드백 루프를 만듭니다.

## 항목 검토

| 부동산 | 유형 | 식별자? |
|---|---|---|
| `reviewId` | 문자열 | `rating` |
| `title` | 정수 | |
| `body` | 문자열 | |
| `verified` | 문자열 | |
| ✓ | 부울 | |

`verified` 부울은 리뷰어가 실제로 제품을 구매했는지 여부를 나타냅니다. 이는 다른 구매자와 분석을 위한 중요한 신뢰 신호입니다.

## 새로운 관계

- **작성** — `Buyer` → `Review`(일대다) 구매자는 시간이 지남에 따라 많은 리뷰를 작성할 수 있습니다.

- **리뷰** — `Review` → `Product`(다대일) 각 리뷰는 정확히 하나의 제품에 대한 것이지만, 제품에는 여러 개의 리뷰가 있을 수 있습니다.

> **피드백 루프:** `Buyer → writes → Review → reviews → Product` 경로는 제품으로 돌아가는 순환을 생성합니다. 즉, 구매자가 제품을 소비한 다음 제품을 검토하여 다른 구매자에게 영향을 미칩니다.

## 전체 그래프

*완전한 전자상거래 온톨로지: 5개 엔터티, 6개 관계. 검토를 통해 구매자 피드백 루프가 종료됩니다.*

## 완전한 모델이 가능하게 하는 것

| 질문 | 그래프 경로 |
|---|---|
| 검증된 리뷰가 가장 높은 평가를 받은 제품은 무엇입니까? | 리뷰(확인=true) → 제품 |
| 장바구니가 가득 차 있지만 주문이 없는 구매자는 누구입니까? | 구매자 → 구매자가 없는 장바구니(itemCount > 0) → 주문 |
| 카테고리 내 제품의 평균 평점은 얼마입니까? | 리뷰 → 제품(카테고리별 그룹) |
| 어떤 충성도 높은 구매자가 가장 많은 리뷰를 작성합니까? | 구매자(loyaltyTier=Gold) → 검토(개수) |

## GQL 쿼리 예

현재 다른 사람의 장바구니에 있는 제품에 대한 검증된 리뷰를 찾으세요.

```gql
MATCH (b:Buyer)-[:has_cart]->(c:Cart)-[:contains]->(p:Product)<-[:reviews]-(r:Review)
WHERE r.verified = true
RETURN p.name, r.rating, r.title
```

## 우리가 만든 것

| 단계 | 추가된 엔터티 | 누적 | 핵심 개념 |
|---|---|---|---|
| 1 | 구매자, 제품, 주문 | 3 | 구매 흐름, SKU 식별자 |
| 2 | 장바구니 | 4 | 세션 엔터티, 일대일 |
| 3 | 검토 | 5 | 피드백 루프, 검증된 신뢰 |

## 주요 내용

1. **세션 엔터티**(카트) 진행 중 상태 캡처
2. **일대일** 관계 모델 독점 소유권
3. **부울 속성**(검증됨)은 신뢰 기반 필터링을 활성화합니다.
4. **피드백 루프**는 선형 체인보다 더 풍부한 쿼리 경로를 생성합니다.
5. 전체 그래프를 통해 탐색부터 검토까지 **퍼널 분석**이 가능합니다.

```quiz
Q: 리뷰 엔터티가 이 온톨로지에서 "피드백 루프"를 생성하게 만드는 이유는 무엇입니까?
- 그래프의 다른 모든 엔터티에 연결됩니다.
- 구매 경로와 다른 경로를 통해 구매자에서 제품으로 돌아가는 경로를 생성합니다. [correct]
- 어떤 개체보다 가장 많은 속성을 가지고 있습니다.
- 불리언 검증 속성을 사용합니다.
> Review가 없으면 Buyer에서 Product까지의 경로는 Order로만 이루어집니다. 리뷰는 구매자 → 리뷰 → 제품이라는 두 번째 경로를 생성하여 루프를 형성합니다. 이 이중 경로 구조는 비교 쿼리(예: "구매했지만 리뷰하지 않음"과 "검토했지만 구매하지 않음")를 가능하게 합니다.
```
