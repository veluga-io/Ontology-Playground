## 조직 백본 구축

모든 HR 온톨로지는 세 가지 핵심 엔터티로 시작됩니다.

- **직원** — 직원
- **부서** — 업무가 조직되는 사업 단위
- **직위** — 책임과 수준을 설명하는 역할 정의

이 세 엔터티는 채용, 보고 및 인력 계획을 위한 최소 구조를 제공합니다.

직위는 현재 할당된 사람과 역할 정의를 분리합니다.

## 이 분리가 중요한 이유

이러한 개념을 하나의 "EmployeeProfile" 엔터티로 축소하면 다음에 대한 유연성이 상실됩니다.

- 역사적인 직원 배치 변경
- 역할 전환
- 고용되기 전에 존재했던 공석

별도의 엔터티는 모델을 깔끔하고 확장 가능하게 유지합니다.

- 식별자 속성을 사용하지 않으려면
