## 인력 이력 문제

직원은 시간이 지남에 따라 부서나 직위 간에 이동할 수 있습니다. 부서는 많은 직원을 수용할 수 있습니다. 시간이 지나면서 다른 사람들이 한 자리를 채울 수 있습니다.

이는 단순한 일대일 구조가 아닙니다.

## 접합 엔터티로 지정

연결하려면 **과제**를 만드세요.

- `Employee` -> `Assignment` (일대다)
- `Assignment` -> `Department` (다대일)
- `Assignment` -> `Position` (다대일)

할당은 관계의 컨텍스트를 유지합니다.

### 할당 속성

| 부동산 | 유형 | 식별자? |
|---|---|---|
| `assignmentId` | 문자열 | `startDate` |
| `endDate` | 날짜 | |
| `isPrimary` | 날짜 | |
| ✓ | 부울 | |

`startDate` 및 `endDate`를 사용하면 다음과 같은 역사적 질문에 답할 수 있습니다.

- "2분기에 재무 부서에 누가 있었나요?"
- "올해 부서를 바꾼 직원은 누구인가요?"

## 실제 디자인 패턴

이는 많은 도메인에서 사용되는 것과 동일한 일반적인 패턴입니다.

- 등록을 통한 학생 과정
- 주문 라인 항목을 통한 고객-제품
- 배정을 통한 사원-부서-직위

관계에 고유한 속성이 필요한 경우 접합 엔터티를 사용합니다.

```quiz
Q: 할당이 자체 엔터티여야 하는 주된 이유는 무엇입니까?
- 그래프의 아이콘 선택이 향상되었습니다.
- startDate 및 endDate와 같은 관계별 속성을 전달합니다. [correct]
- 식별자가 필요하지 않습니다.
- 다대일 관계를 방지합니다.
> 할당은 시간 경과에 따른 직원 채용 상황을 저장합니다. 이러한 속성은 직원, 부서 또는 직위에만 속하지 않고 관계에 속합니다.
```
