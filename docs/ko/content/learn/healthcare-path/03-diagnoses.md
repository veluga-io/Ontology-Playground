## 임상 결과 기록

예약을 하면 임상 결과가 나옵니다. 환자의 상태는 어떤가요? **진단** 엔터티는 표준화된 코딩을 통해 이러한 결과를 캡처합니다.

진단을 추가하면 다음이 가능해집니다.
- "당뇨병 진단을 받은 환자는 누구인가?"
- "지난 분기에 가장 심각한 상황을 확인한 공급자는 누구입니까?"
- "부서별로 가장 많이 발생하는 진단은 무엇인가요?"

## 진단 엔터티

| 부동산 | 유형 | 식별자? |
|---|---|---|
| `diagnosisId` | 문자열 | `icdCode` |
| `description` | 문자열 | |
| `severity` | 문자열 | |
| `diagnosedDate` | 문자열 | |
| ✓ | 날짜 | |

`icdCode` 속성은 세계적으로 인정받는 코딩 시스템인 표준화된 ICD(국제 질병 분류) 코드를 보유하고 있습니다. 이는 온톨로지를 보험, 청구 및 연구 시스템과 상호 운용 가능하게 만듭니다.

## 새로운 관계

- **diagnosed_with** — `Patient` → `Diagnosis`(일대다) 환자는 병력에 걸쳐 여러 진단을 받을 수 있습니다.

- **진단** — `Provider` → `Diagnosis`(일대다) 서비스 제공자는 임상 평가를 기반으로 진단을 기록합니다.

> **이중 저자:** 진단은 환자(질환이 있는 사람)와 제공자(질환을 식별한 사람) 모두에게 연결됩니다. 이 이중 연결을 통해 환자 중심 보기("내 모든 조건")와 제공자 중심 보기("내가 식별한 모든 조건")가 모두 가능합니다.

*진단은 환자와 제공자 모두에 대한 연결과 함께 그래프에 합류합니다. 차이점은 새로운 기능을 강조합니다.*

## 우리가 배운 것

- **표준화된 코드**(ICD)는 온톨로지를 외부 시스템과 상호 운용 가능하게 만듭니다.
- **이중 연결 개체**(환자 → 진단 ← 제공자) 두 관점을 모두 포착
- **심각도 속성**을 통해 위험 계층화 및 임상 우선순위 지정 가능
- 이제 그래프는 예약 쿼리(예약을 통해)와 임상 쿼리(진단을 통해)를 모두 지원합니다.

```quiz
Q: 진단 엔터티에 ICD 코드 속성이 중요한 이유는 무엇입니까?
- 진단 식별자를 짧게 만듭니다.
- 보험, 청구, 연구 시스템과의 상호운용성을 가능하게 하는 글로벌 표준화된 코딩 시스템을 제공합니다. [correct]
- 모든 온톨로지 형식에는 ICD 코드가 필요합니다.
- 중복 진단이 기록되는 것을 방지합니다.
> ICD(국제질병분류) 코드는 질병을 분류하는 보편적인 표준입니다. 이를 온톨로지에 포함하면 상호 운용성이 가능해집니다. 동일한 코드는 EHR, 보험 청구, 임상 시험 및 공중 보건 시스템 전반에 걸쳐 동일한 조건을 의미합니다.
```
